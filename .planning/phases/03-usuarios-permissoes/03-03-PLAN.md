    **AvatarService** (`app/Services/AvatarService.php`):

    Namespace `App\Services`. Uses `App\Models\User`, `Illuminate\Http\UploadedFile`, `Illuminate\Support\Facades\Storage`.

    Constants:
    - `private const SIZE = 256`
    - `private const DISK = 'public'`
    - `private const DIRECTORY = 'avatars'`

    Methods:

    1. `store(User $user, UploadedFile $file): string`
       - Delete existing avatar: `$this->deleteExisting($user)`
       - Process image using Laravel 13 Image API:
         ```php
         $path = $file->image()
             ->cover(self::SIZE, self::SIZE)
             ->toWebp(quality: 80)
             ->storePublicly(path: self::DIRECTORY, disk: self::DISK);
         ```
       - If `!$path`: throw `RuntimeException('Falha ao armazenar avatar.')`
       - Update user: `$user->forceFill(['avatar_path' => $path])->save()`
       - Return `$path`
       - Note: Laravel 13 Image API is available natively. If `$file->image()` throws "method not found", the Image API may need to be enabled. Verify in `composer.json` that `laravel/framework ^13.8` is used. If the method does not exist, use the Intervention Image facade directly: `Image::read($file)->cover(256, 256)->toWebp(80)->storePublicly('avatars', 'public')`. The first-party Image helper was added to Laravel 13 in late 2025.

    2. `deleteExisting(User $user): void`
       - If `$user->avatar_path` is not null and `Storage::disk(self::DISK)->exists($user->avatar_path)`:
       - Delete: `Storage::disk(self::DISK)->delete($user->avatar_path)`

    3. `url(User $user): ?string`
       - Return `$user->avatar_path ? Storage::disk(self::DISK)->url($user->avatar_path) : null`

    4. `deleteByPath(string $path): void`
       - Helper: `Storage::disk(self::DISK)->delete($path)` if file exists

    **UpdateProfileRequest** (`app/Http/Requests/UpdateProfileRequest.php`):

    Namespace `App\Http\Requests`. Extend `FormRequest`.

    - `authorize()`: return `$this->user() !== null` (any authenticated user)
    - `rules()`:
      - `name`: sometimes, string, max:255
      - `email`: sometimes, email, unique:users,email,{$this->user()->id}
      - `phone`: nullable, string, max:20
      - `position`: nullable, string, max:100
      - `department`: nullable, string, max:100
      - `signature`: nullable, string, max:255
    - `messages()`: Portuguese error messages

    **StoreAvatarRequest** (`app/Http/Requests/StoreAvatarRequest.php`):

    - `authorize()`: return `$this->user() !== null`
    - `rules()`:
      - `avatar`: required, image, mimes:jpeg,png,webp, mimetypes:image/jpeg,image/png,image/webp, max:2048 (2MB), dimensions:min_width=128,min_height=128
    - `messages()`:
      - `avatar.required`: 'Avatar é obrigatório.'
      - `avatar.image`: 'O arquivo deve ser uma imagem.'
      - `avatar.mimes`: 'Formatos aceitos: JPEG, PNG, WebP.'
      - `avatar.max`: 'A imagem deve ter no máximo 2MB.'
      - `avatar.dimensions`: 'A imagem deve ter no mínimo 128x128 pixels.'

    The `authorize()` method returns `true` effectively since we check `$this->user() !== null` — which means authenticated users are always authorized. The actual permission check (user can only edit their own profile) is handled in the ProfileController.

    **ProfileController** (`app/Http/Controllers/Api/V1/ProfileController.php`):

    Namespace `App\Http\Controllers\Api\V1`. Use `App\Models\User`, `App\Services\AvatarService`, `App\Http\Requests\UpdateProfileRequest`, `App\Http\Requests\StoreAvatarRequest`.

    Class-level `#[Middleware('auth:sanctum')]` — all profile endpoints require authentication.

    Methods:

    1. `show(Request $request): JsonResponse`
       - Return the authenticated user with roles: `$request->user()->load('roles.permissions')`
       - Include avatar URL: add `'avatar_url' => app(AvatarService::class)->url($request->user())` to the response
       - Return `response()->json($userData)`

    2. `update(UpdateProfileRequest $request): JsonResponse`
       - Get validated data from request
       - Update authenticated user: `$request->user()->update($validated)`
       - Return 200 with updated user and message: `'Perfil atualizado com sucesso.'`

    3. `updatePassword(Request $request): JsonResponse`
       - Validate manually or use inline validation:
         - `current_password`: required, current_password (Laravel 13 rule)
         - `password`: required, string, min:8, confirmed
         - `password_confirmation`: required
       - `$request->user()->update(['password' => Hash::make($request->password)])`
       - Return 200 with message: `'Senha alterada com sucesso.'`
       - Note: Logout from other devices? For Sanctum SPA (session-based), password change invalidates session naturally if `password_updated_at` check is configured. For now, just update and return success.

    4. `updateAvatar(StoreAvatarRequest $request, AvatarService $avatarService): JsonResponse`
       - Get the authenticated user: `$user = $request->user()`
       - Call `$avatarService->store($user, $request->file('avatar'))`
       - Return fresh user data with new avatar_url:
         ```php
         $user->fresh();
         return response()->json([
             'user' => $user->load('roles.permissions'),
             'avatar_url' => $avatarService->url($user),
             'message' => 'Avatar atualizado com sucesso.',
         ]);
         ```

    5. `deleteAvatar(AvatarService $avatarService): JsonResponse`
       - Delete avatar: `$avatarService->deleteExisting($request->user())`
       - Update user: `$request->user()->forceFill(['avatar_path' => null])->save()`
       - Return 200: `'Avatar removido com sucesso.'`

    **Routes** (`routes/api.php`):

    Add inside the existing `auth:sanctum` middleware group (or create a new group):

    ```php
    use App\Http\Controllers\Api\V1\ProfileController;

    Route::middleware('auth:sanctum')->group(function () {
        // ... existing routes ...
        Route::get('profile', [ProfileController::class, 'show'])->name('profile.show');
        Route::put('profile', [ProfileController::class, 'update'])->name('profile.update');
        Route::put('profile/password', [ProfileController::class, 'updatePassword'])->name('profile.password');
        Route::post('profile/avatar', [ProfileController::class, 'updateAvatar'])->name('profile.avatar.update');
        Route::delete('profile/avatar', [ProfileController::class, 'deleteAvatar'])->name('profile.avatar.delete');
    });
    ```

    If the `auth:sanctum` group already exists (from Plan 01 user/role routes), add these profile routes inside that same group.
