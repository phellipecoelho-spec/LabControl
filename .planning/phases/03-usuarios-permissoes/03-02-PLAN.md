    **stores/users.ts:**

    Create Pinia store `useUsersStore` following the pattern from `stores/auth.ts`.

    State:
    - `users: Ref<User[]>` (initial `[]`)
    - `loading: Ref<boolean>` (initial `false`)
    - `pagination: Ref<{current_page, last_page, total, per_page}>` (initial `{...}`)

    Actions:
    - `async fetchAll(params?: {search?, role?, status?, page?})` — calls `api.get('/users', { params })`, updates `users` and `pagination`. Handle 403 by returning early.
    - `async fetchById(id: string)` — calls `api.get('/users/${id}')`
    - `async create(data)` — calls `api.post('/users', data)`, returns response data
    - `async update(id: string, data)` — calls `api.put('/users/${id}', data)`, returns response data. After update, refresh the user in local `users` array.
    - `async destroy(id: string)` — calls `api.delete('/users/${id}')`, removes user from local array

    Type imports: `import type { User } from '@/stores/auth'` for the User interface. Extend User type locally if needed (phone, position, department).

    **stores/roles.ts:**

    Create Pinia store `useRolesStore`.

    State:
    - `roles: Ref<Role[]>` with interface `Role` having `{id, name, slug, description, is_system, permissions: Permission[], users_count?: number}`
    - `loading: Ref<boolean>`

    Actions:
    - `async fetchAll(params?: {with_users?})` — calls `api.get('/roles', { params })`
    - `async fetchById(id: string)` — calls `api.get('/roles/${id}')`
    - `async create(data)` — calls `api.post('/roles', data)`
    - `async update(id: string, data)` — calls `api.put('/roles/${id}', data)`
    - `async syncPermissions(id: string, permissionIds: string[])` — calls `api.post('/roles/${id}/permissions', { permissions: permissionIds })`, then refreshes the specific role in `roles` array
    - `async destroy(id: string)` — calls `api.delete('/roles/${id}')`

    **services/api.ts:**

    No changes needed — the existing `api` instance with `baseURL: '/api/v1'` handles all new endpoints automatically. The store actions use `api.get('/users')` which resolves to `/api/v1/users`. Verify the baseURL ends with `/api/v1` (yes, confirmed from existing file). If new endpoints need to be documented, they follow the pattern `api.get('/users')`.
