        - Para cada usuário, criar registro na tabela `notifications`:
          ```php
          DB::table('notifications')->insert([
              'id' => (string) \Illuminate\Support\Str::uuid(),
              'type' => 'App\Notifications\LoanOverdue',
              'notifiable_type' => 'App\Models\User',
              'notifiable_id' => $user->id,
              'data' => json_encode([
                  'loan_id' => $loan->id,
                  'borrower_name' => $loan->borrower->name,
                  'equipment_count' => $loan->equipment->count(),
          ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API Request → LoanController | Dados HTTP não confiáveis entram no sistema via Form Request |
| LoanController → LoanService | Chamada de serviço com dados validados |
| Command → Database | Scheduled command acessa DB para criar notificações |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-04 | Spoofing | LoanController actions | high | mitigate | Permission middleware emprestimos.{view,create,edit,finalizar} em cada action. Usuário não autenticado não acessa nenhum endpoint. |
| T-07-05 | Tampering | StoreLoanRequest | high | mitigate | Validação de `equipment_ids` array, `borrower_id` exists:users, datas consistentes (expected_return_at > borrowed_at). Verificação de conflito no LoanService. |
| T-07-06 | Tampering | returnItem action | medium | mitigate | ReturnLoanItemRequest valida equipment_id e pertence ao loan. LoanService valida que loan.status = active. Apenas equipamentos do loan podem ser devolvidos. |
| T-07-07 | Information Disclosure | Loan list/index | low | accept | Apenas dados de negócio (empréstimos) expostos — sem dados sensíveis como senhas. Permissão emprestimos.view exigida. |
| T-07-08 | Denial of Service | Scheduled command | low | accept | Command roda 1x/dia via schedule. Em caso de muitos overdue loans, as notificações podem ser muitas, mas são inserções simples no DB sem impacto significativo. |
</threat_model>
