## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Label | 13px | 600 | 1.4 | Sidebar module labels, form field labels, badge text, compact table cells |
| Body | 14px | 400 | 1.5 | Primary reading text, paragraph content, descriptions, card body content |
| Heading | 20px | 600 | 1.3 | Page titles, section headers, dialog titles, panel headers, card titles |
| Display | 28px | 600 | 1.2 | Dashboard metric numbers, hero stats, large data-points |

**Size consolidation note:** 6 sizes reduced to 4 per checker constraint. Card titles and subtitles use Body size (14px) with semibold (600) weight — a standard pattern used by Linear, shadcn, and Vercel design systems. Page titles (formerly 24px) and Section titles (formerly 20px) merged into Heading role at 20px; the size difference from Body (14px → 20px) provides sufficient visual hierarchy. Label (13px) retained for sidebar compact state where 14px is too wide.

**Weight consolidation note:** 4 weights reduced to 2 per checker constraint. Only regular (400) and semibold (600) are used. Label, Heading, and Display all use 600 (semibold); visual distinction comes from size differences (13 → 20 → 28) rather than weight gradation.

**Notes:**
- PrimeVue Aura theme default font size is 14px — body text maps directly
- Sidebar PanelMenu items use the Label role (13px, 600)
- Sidebar category headers use Heading role (20px, 600) with uppercase styling
- Card titles within the layout (e.g., user menu card) use Body size (14px) with semibold (600)

## Interaction Contracts

### 1. Sidebar Collapse (Desktop)

| Property | Value |
|----------|-------|
| Expanded width | 240px |
| Collapsed width | 64px |
| Animation | CSS transition 0.2s ease (width + opacity on labels) |
| Toggle trigger | Hamburger icon in topbar, leftmost position |
| Expanded state | Icon + text label visible, category headers visible, group spacing maintained |
| Collapsed state | Icon only (centered), text labels hidden with `overflow: hidden`, category headers hidden, PanelMenu groups collapsed to single icon row |
| Transition target | Sidebar element only — content area reflows via `grid-template-columns` or margin-left |
| PanelMenu in collapsed | Show only root icon. On hover/focus, show contextual tooltip with module name. |

### 2. Mobile Drawer (Viewport < 768px)

| Property | Value |
|----------|-------|
| Component | PrimeVue Drawer (overlay mode, not permanent) |
| Width | 300px (wider than desktop expanded to accommodate touch targets) |
| Position | Left |
| Backdrop | Semi-transparent black `rgba(0,0,0,0.5)` with `z-index` above topbar |
| Toggle trigger | Hamburger icon in topbar |
| Close trigger | Click outside drawer (backdrop), close (X) icon in drawer header, swipe right-to-left |
| Drawer header | App logo/name "LabControl" + close button |
| Content | Full PanelMenu navigation (same as desktop expanded) |
| Animation | PrimeVue Drawer default slide-in from left (0.3s) |
| Body scroll | Locked when drawer open |

### 3. Topbar Behavior

| Element | Interaction |
|---------|-------------|
| Hamburger toggle | Leftmost — toggles sidebar collapse (desktop) or opens drawer (mobile) |
| App title/logo | Center or next to hamburger — always visible. Click navigates to `/` (dashboard) |
| Theme toggle | Right side — sun icon (light) / moon icon (dark). Click toggles `.app-dark` class. No animation transition needed (PrimeVue Aura handles instantly). |
| Notifications bell | Next to theme toggle — bell icon with `p-badge` showing "0" (placeholder). Click: no action in this phase (future: opens notification panel). |
| User avatar menu | Rightmost — PrimeVue Avatar + downward chevron. Click opens overlay panel with: "Meu Perfil" link (→ `/profile`), divider, "Sair" (calls `authStore.logout()`). |
| Profile menu item | Icon: `pi pi-user`. Click navigates to profile. |
| Logout menu item | Icon: `pi pi-sign-out`. Click triggers logout. No confirmation dialog — logout is reversible (relogin). |

### 4. Theme Toggle

| Property | Value |
|----------|-------|
| Persistence | `localStorage` key `app-theme` with values `"dark"` / `"light"` |
| Default | `"dark"` (if no localStorage value found) |
| Mechanism | Toggle `.app-dark` class on `document.documentElement` |
| PrimeVue integration | Already configured via `darkModeSelector: '.app-dark'` in `main.ts` |
| OS preference | NOT implemented per D-12 — manual toggle only |
| Icon | Moon `pi pi-moon` when light mode (click switches to dark); Sun `pi pi-sun` when dark mode (click switches to light) |
| Transition | Instant (no CSS transition on body background) |

**Integration:** `App.vue` uses a computed wrapper based on route meta:

## Route Meta Contract

All authenticated routes MUST include the following `meta` fields for sidebar highlighting and module tracking:

```typescript
interface RouteMeta {
  requiresAuth: boolean
  title: string           // Page title displayed in topbar/content
  module?: string         // Module slug for sidebar active state (e.g. 'admin.users', 'dashboard')
  roles?: string[]        // Permission roles (optional)
  guest?: boolean         // Public route flag (optional)
}
```

**Required updates to existing routes:**
- Dashboard route: add `module: 'dashboard'`
- Admin routes: add `module: 'admin.users'`, `module: 'admin.roles'`, `module: 'admin.logs'`
- Profile route: add `module: 'profile'`

**Source:** CONTEXT.md — code_context integration points with router/routes.ts. PanelMenu active state detection requires consistent `module` meta.

## Responsive Breakpoints

| Breakpoint | Width | Layout |
|------------|-------|--------|

| Tablet | 768px – 1023px | Same as desktop (sidebar collapsible) |

**PrimeVue responsiveness:** Use PrimeVue Drawer with `dismissable` and `position="left"` for mobile. Use CSS or computed class based on `window.innerWidth` or `@vueuse/core`'s `useMediaQuery` for breakpoint detection.

## Accessibility & UX Notes

- All sidebar icons must have `aria-label` matching the module label
- Sidebar toggle button must have `aria-expanded` reflecting collapse state
- Theme toggle must have `aria-label` reflecting current mode
- PanelMenu must be keyboard-navigable (PrimeVue default behavior — verify)
- Focus ring color: use accent (#6366f1) with 2px offset
- Touch targets in mobile drawer: minimum 44px height (PrimeVue PanelMenu default meets this)
- Collapsed sidebar icons must remain visible and clickable — only labels hide
- Skip-to-content link: implement visually hidden skip link as first focusable element in AppLayout
