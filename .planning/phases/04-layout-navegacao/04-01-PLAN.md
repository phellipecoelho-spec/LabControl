  <action>
    Create `frontend/src/composables/useTheme.ts` with the following implementation:

    1. **`STORAGE_KEY` constant** ŌĆö `'app-theme'` (per UI-SPEC "Theme Toggle" section)

    2. **`isDark` ref** ŌĆö Initialized from `localStorage.getItem(STORAGE_KEY) !== 'light'` (dark default per D-10)

    3. **`applyTheme()` function** ŌĆö Toggles `.app-dark` class on `document.documentElement`:
       ```typescript
       function applyTheme() {
         const root = document.documentElement
         root.classList.toggle('app-dark', isDark.value)
       }
       ```
       Per D-13 and RESEARCH.md Pitfall 4, MUST use `document.documentElement`, never `document.body`.

    4. **`toggle()` function** ŌĆö Flips `isDark.value`. No parameters.

    5. **`watch(isDark, ...)`** ŌĆö Persists to localStorage and calls `applyTheme()` on change. Uses `{ immediate: true }` to apply on init.

    6. **Return** `{ isDark: Ref<boolean>, toggle: () => void }` ŌĆö `isDark` is readonly in return (use `readonly(isDark)` to prevent external mutation).

    Per D-12: Do NOT implement `prefers-color-scheme` detection. Manual toggle only.

    Per D-10: Default is dark mode. The `'light'` localStorage value is the only way to switch to light.

    Follow the exact function signatures from RESEARCH.md Pattern 3. The composable must be callable from any component via `const theme = useTheme()`.
  </action>
