    ### B) Update `frontend/src/App.vue`

    **Replace existing template** with conditional layout pattern (per RESEARCH.md Pattern 1):
    ```vue
    <template>
      <Toast position="top-right" />
      <AppLayout v-if="route.meta.requiresAuth">
        <router-view />
      </AppLayout>
      <router-view v-else />
    </template>

    <script setup lang="ts">
    import { useRoute } from 'vue-router'
    import Toast from 'primevue/toast'
    import AppLayout from '@/components/layout/AppLayout.vue'

    const route = useRoute()
    </script>
    ```

    **Logic:**
    - `route.meta.requiresAuth === true` → wrap `<router-view />` in `<AppLayout>` (shell)
    - All other cases (guest routes, no meta) → render `<router-view />` directly (no shell)

    This ensures auth pages (login, register, forgot-password, reset-password, verify-email) render WITHOUT the layout shell per UI-SPEC "Route Guard & Layout Mapping" and CONTEXT.md "Auth views (exclusoes do layout)".
